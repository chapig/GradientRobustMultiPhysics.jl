Abstract Actions

General concept

Parameters (possibly region- or item-dependent) are handle by AbstractAction types and can be assigned to Assembly Patterns. The action is applied to some input vector (usually the first argument of some assembly pattern) and then returns the modified result. Each AbstractAction implements the apply! interface

apply!(result,input,::AbstractAction,qp::Int)

that defines how the result vector is computed based on the input and the current quadrature point number.

Before it is applied to any input or quadraturepoint, an update! step is performed upon entry on any item which prepares the data for the evaluation (fixing item and region numbers or computing the global coordinates of the quadrature points). Details of that are hidden here, since it is usually not necessary for the user to interfere with that.

Below is a table with implemented action types and some short description what it can be used for. Click on the action name for more details.

Action typeWhat it does
DoNotChangeActiondoes nothing (result = input)
MultiplyScalarActionmultiplies argument (vector) with a scalar
RegionWiseMultiplyScalarActionmultiplies argument (vector) with a scalar that depends on the region
MultiplyVectorActionmultiplies argument (vector) componentwise with a vector of same length
RegionWiseMultiplyVectorActionmultiplies argument (vector) componentwise with a region-depending vector of same length
MultiplyMatrixActionmatrix-vector multiplication, input is multiplied with the specified matrix
FunctionActionf!(result, input) for some user-specified function with this interface
XFunctionActionf!(result, input, x) for some user-specified function (that also depends on x)
ItemWiseXFunctionActionf!(result, input, x, item) for some user-specified function (that also depends on x and item number)
RegionWiseXFunctionActionf!(result, input, x, region) for some user-specified function (that also depends on x and region id)

GradientRobustMultiPhysics.FunctionAction — Type
struct FunctionAction{T<:Real} <: AbstractAction

action that puts input into the specified function f! and returns its result. The function f! should have the interface

f!(result,input)

and result is expected to be of length resultdim.

The quadrature order of assemblies that involve this action can be altered with bonus_quadorder.
